## SQL a relační databáze
- SQL developed at IBM by Chamberlin & Boyce (SEQUEL paper): 1974
- Oracle releases first commercial relational DB: 1979
- IBM SQL/DS: 1981; IBM DB2: 1983
- SQL becomes ANSI/ISO standard: 1986 — https://en.wikipedia.org/wiki/SQL
- MySQL: 1995; PostgreSQL (freeware): 1996
- ORM frameworks reduce direct SQL: Hibernate 2001, ActiveRecord (Rails) 2004, Entity Framework 2008
- NoSQL movement: MongoDB 2009, Redis 2009, Cassandra 2008
  - https://en.wikipedia.org/wiki/NoSQL
- Cloud managed databases (RDS, Cloud SQL, Aurora) shift expectation away from raw SQL tuning
- Stack Overflow 2023: SQL still 3rd most used language (48%) but role shifting to ORMs/ORM-first

## Desktop / nativní GUI
- Xerox Alto GUI: 1973 (research); Xerox Star (commercial): 1981
- Apple Lisa: 1983; Apple Macintosh: 1984 — https://en.wikipedia.org/wiki/Apple_Macintosh
- Windows 1.0: 1985; Windows 3.0: 1990 (first widely adopted); Windows 3.1: 1992
- Win32 API: Windows NT 3.1 (1993); MFC 1.0: 1992
- Delphi 1.0: 1995 (rapid GUI development for Windows)
- Windows 95: August 1995 — peak of desktop GUI era
- Web apps (CGI, then ASP, PHP) begin competing: 1996–2000
- iPhone: 2007 — decisive shift to mobile/web paradigms
- Desktop GUI as mainstream senior expectation collapses ~2004–2010
- Electron (web-tech desktop): 2013 — minor partial recovery but not traditional GUI
  - https://en.wikipedia.org/wiki/Electron_(software_framework)

## Design patterns (GoF)
- Christopher Alexander's "A Pattern Language": 1977 (original inspiration)
- Kent Beck & Ward Cunningham introduce patterns to software: 1987 (OOPSLA paper)
- "Design Patterns: Elements of Reusable Object-Oriented Software" (GoF): October 1994
  - Authors: Gamma, Helm, Johnson, Vlissides
  - https://en.wikipedia.org/wiki/Design_Patterns
- Java (1995) and its enterprise ecosystem popularized GoF patterns as near-mandatory vocabulary
- J2EE patterns proliferate: 1999–2005 (DAO, Facade, Singleton everywhere)
- Functional programming pushback against OOP pattern overuse: ~2010–2015
  - "Simple Made Easy" Rich Hickey (2011); "Why OO Sucks" Joe Armstrong
- Modern decline: microservices, functional idioms, and LLMs reduce need for rote pattern knowledge
- GoF book added to KEY_EVENTS annotation on chart at 1994

## Vícevláknové programování
- POSIX threads (pthreads) IEEE standard draft: 1988; finalized POSIX.1c: 1995
  - https://en.wikipedia.org/wiki/Pthreads
- OS/2 multithreading: 1987; Windows NT preemptive multithreading: 1993
- Java Thread API: 1995 (built-in, widely taught)
- Intel Pentium D (first consumer dual-core): 2005 — concurrency becomes mandatory
- Intel Core 2 Quad (quad-core): 2007 — "concurrency crisis" peaks
- Go goroutines: 2009; async/await: C# 2012, Python 3.4 2014, JavaScript (ES2017) 2017
- Reactive frameworks (RxJava 2013, Akka, Project Reactor) abstract raw threading
- Modern async patterns largely replace manual thread management for most senior devs

## Version Control History (Verzování skill)
- SCCS (Source Code Control System): developed 1972 at Bell Labs by Marc Rochkind;
  distributed with UNIX in Programmer's Workbench (PWB/UNIX) from 1977
  - https://en.wikipedia.org/wiki/Source_Code_Control_System
  - https://machaddr.substack.com/p/the-history-and-influence-of-sccs
- RCS (Revision Control System): released 1982 by Walter Tichy (Purdue Univ.);
  replaced SCCS as Unix standard through mid-1980s
  - https://en.wikipedia.org/wiki/Revision_Control_System
  - https://experimentalworks.net/posts/2024-03-18-a-history-of-vcs-part1/
- CVS (Concurrent Versions System): v0 by Dick Grune 1986; CVS 1.0 by Brian Berliner 1990;
  dominant in open source (GNU, Linux) through late 1990s–early 2000s
  - https://en.wikipedia.org/wiki/Concurrent_Versions_System
- SVN (Subversion): released 2000; became most popular centralized VCS by ~2006;
  supplanted CVS in enterprise and many open source projects
  - https://en.wikipedia.org/wiki/Apache_Subversion
- Git: created by Linus Torvalds, first released April 6, 2005 (Linux kernel)
  - GitHub launched April 2008 — catalyzed mainstream Git adoption
- Stack Overflow VCS adoption data:
  - 2015: Git 69.3%, SVN 36.9% — https://survey.stackoverflow.co/2015
  - 2018: Git 87%, SVN 16% — https://survey.stackoverflow.co/2018
  - 2022: Git 93%+ — https://survey.stackoverflow.co/2022/
  - 2025: Git 93.87% — https://survey.stackoverflow.co/2025/
  - Detailed analysis: https://stackoverflow.blog/2023/01/09/beyond-git-the-other-version-control-systems-developers-use/
- Bitbucket 2020: announced end of Mercurial support, citing 90% of devs use Git, <1% new projects use Mercurial
- History overview: https://dev.to/thefern/history-of-version-control-systems-vcs-43ch
- https://initialcommit.com/blog/Technical-Guide-VCS-Internals

# Corrections applied to skills-timeline.json

## Automatizované testování (Automated testing)
- PROBLEM: Original curve started rising in 1985; value of 5.5 in 2000 was too high.
- FIX: Curve now flat at 2.0 until 1996. Rise begins 1997 (JUnit creation).
  Midpoint shifted to 2008 (Agile mainstream). 2000 corrected from 5.5 → 2.4.
- RATIONALE: JUnit: 1997. TDD book (Kent Beck): 2002. Agile mainstream: 2006-2010.
  Senior dev expectation of automated testing solidified ~2014-2018.

## Refaktoring kódu (Code refactoring)
- PROBLEM: Original curve started rising in 1990; value of 4.61 in 2000 was too high.
- FIX: Curve flat at 3.0 until 1998. Rise begins 1999 (Fowler's "Refactoring" book).
  Midpoint shifted to 2007. 2000 corrected from 4.61 → 3.48.
- RATIONALE: Fowler's book (1999) established the vocabulary and catalog.
  IDE automated refactoring: IntelliJ 2001, Eclipse 2001-2004.
  Agile/XP practices drove mainstream adoption 2004-2010.

## Observabilita systému (System observability)
- PROBLEM: Original curve started rising in 2005 and reached 5.5 by 2015 — too high.
  "Three pillars" (logs+metrics+traces) concept only articulated ~2016-2017.
- FIX: Flat at 1.0 until 2009. Rise begins 2010 (Graphite/StatsD era).
  Midpoint shifted to 2018 (OpenTelemetry formation year).
  2015 corrected from 5.5 → 3.23; 2018 corrected from 7.11 → 5.5.
- RATIONALE: Prometheus: 2012 (open-sourced 2015). Google SRE book: 2016.
  "Three pillars" term: 2016-2017. OpenTelemetry: 2019.
  Observability as mainstream senior expectation: ~2019-2022.

## Prompt engineering
- PROBLEM: 2024 value of 4.0 and 2026 value of 6.15 underestimated rapid adoption.
- FIX: 2022: 1.85→2.5, 2023: 2.83→3.75, 2024: 4.0→5.0, 2025: 5.17→6.5, 2026: 6.15→7.5
- RATIONALE: ChatGPT launched Nov 2022; GitHub Copilot GA June 2022.
  SO 2024: 62% devs using AI tools. SO 2025: ~80%+ adoption.
  Prompt engineering increasingly expected of senior devs by 2025-2026.

## Validace AI výstupů (AI output validation)
- PROBLEM: 2024 value of 4.5 and 2026 value of 6.92 too low given trust gap data.
- FIX: 2022: 2.08→3.0, 2023: 3.19→4.5, 2024: 4.5→6.0, 2025: 5.81→7.5, 2026: 6.92→8.5
- RATIONALE: SO 2024: AI trust at only 43% despite 62% adoption.
  SO 2025: trust declining further despite growing adoption.
  Critically reviewing AI-generated code is rapidly becoming a must-have senior skill.

## Verzování (Version Control) — replaces "Distribuované verzování Git"
- PROBLEM: Original curve was Git-specific, starting at 1.0 through 2004 then rising from 2005.
  This erased 30+ years of VCS history (SCCS, RCS, CVS, SVN).
- FIX: Renamed to "Verzování". Curve now starts rising from 1972 (SCCS), accelerates through
  CVS era (1990: 4.5), SVN era (2000: 7.3), reaches near-universal by 2015 (9.9) and 10 by 2016.
- RATIONALE: RCS (1982) established VCS as expected practice in Unix/academic environments.
  CVS (1990) brought it to open source. SVN (2000) made it a professional standard.
  Git (2005) + GitHub (2008) completed the transition to universal adoption.
  SO 2015: Git 69%, SO 2018: 87%, SO 2022: 93%+.

## Hardware category — split from Technical Core
- PROBLEM: "Hardwarové uvažování" and "Práce s pamětí" are hardware-specific skills
  that do not belong in the general Technical Core category alongside language mastery,
  debugging, testing etc.
- FIX: Created new "Hardware" category (color #b0c4de / light steel blue).
  Moved both skills to "hardware".
- RATIONALE: Both skills track the decline of hardware awareness as managed runtimes
  (JVM, .NET CLR, V8) and high-level languages abstracted away memory and hardware details.
  Isolating them visually makes the decline trend clearer.

## Programování category — AI-era decline corrections (2023–2026)
- RATIONALE: SO 2024: 62% of devs use AI tools but only 43% trust AI accuracy.
  SO 2025: adoption ~80%, trust continuing to decline. This trust gap means senior
  developers are expected to EVALUATE AI output rather than write everything from scratch,
  reducing the absolute requirement level for implementation skills.
- Mistrovství syntaxe jazyka: 2024: 8.8→8.5, 2025: 8.3→7.8, 2026: 7.8→7.0
  (syntax generation is LLMs' strongest capability)
- Porozumění sémantice a runtime: 2023: 10→9.8, 2024: 10→9.5, 2025: 10→9.2, 2026: 10→9.0
  (gentlest decline — runtime understanding critical for validating AI-generated code)
- Algoritmické uvažování: 2023: 9→8.7, 2024: 9→8.3, 2025: 9→8.0, 2026: 9→7.5
  (LLMs implement standard algorithms well; evaluative ability remains expected)
- Volba datových struktur: 2023: 8.5→8.2, 2024: 8.5→7.8, 2025: 8.5→7.4, 2026: 8.5→7.0
  (LLMs suggest appropriate structures; senior devs evaluate, not implement from scratch)
- Refaktoring kódu: 2023: 9→8.8, 2024: 9→8.4, 2025: 9→8.0, 2026: 9→7.5
  (AI tools handle mechanical refactoring; architectural judgment remains human)
- Správa závislostí: 2024: 9→8.7, 2025: 9→8.3, 2026: 9→8.0
  (smallest decline — security/compatibility judgment still requires human expertise)
